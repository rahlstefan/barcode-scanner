🚀 БЫСТРЫЙ СТАРТ (5 минут):

Шаг 1: Создайте GitHub репозиторий
   → https://github.com/new
   → Имя: barcode-scanner

Шаг 2: Получите Expo Token
   npm install -g eas-cli
   eas login
   eas credentials

Шаг 3: Добавьте в GitHub Secrets
   Settings → Secrets → New Secret
   Name: EXPO_TOKEN
   Value: <ваш-токен>

Шаг 4: Push кода
   git remote add origin https://github.com/YOUR_USERNAME/barcode-scanner.git
   git add .
   git commit -m "Initial commit"
   git push -u origin main

Шаг 5: Смотрите сборку
   GitHub → Actions → Build iOS with EAS
   Ждите завершения (~10 минут)

Шаг 6: Скачайте .ipa
   https://expo.dev/builds
   Download → .ipa файл готов!

📊 WORKFLOWS - Что они делают:

1️⃣  build-eas.yml (РЕКОМЕНДУЕТСЯ)
    ✅ Автоматический при push на main
    ✅ Использует облако Expo
    ✅ Не требует локального macOS
    ✅ Проще всего настраивать
    📍 .ipa на: https://expo.dev/builds

2️⃣  build-xcode.yml (Альтернатива)
    ⚠️  Только вручную (workflow_dispatch)
    💻 Использует macOS runner GitHub
    ⚠️  Нужны сертификаты Apple
    🔧 Полный контроль
    📍 .ipa на: GitHub Artifacts

3️⃣  build-complete.yml (Продвинутый)
    ✅ Lint → Build → Notify
    ✅ Параллельные jobs
    ✅ Slack уведомления (опционально)
    ✅ Tag-based production builds

4️⃣  test.yml
    ✅ Проверка TypeScript
    ✅ Проверка формата кода
    ✅ Запускается на каждый PR
    ✅ Блокирует merge если ошибки

🐛 TROUBLESHOOTING:

❌ "EXPO_TOKEN not found"
   → GitHub Settings → Secrets → Add EXPO_TOKEN

❌ "Build timed out"
   → EAS Build может быть медленным
   → Проверьте статус на https://expo.dev

❌ "Xcode build failed"
   → Используйте EAS Build вместо Xcode
   → Легче и надёжнее

❌ "Secret not accessible"
   → Убедитесь что вы в правильном репозитории
   → Проверьте названия секретов (case-sensitive)
